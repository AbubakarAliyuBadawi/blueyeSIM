# Blueye Stonefish Simulation

This repository contains a ROS 2 Humble workspace for simulating and controlling the Blueye ROV in Stonefish. It includes the Stonefish simulator assets, the Blueye vehicle model, docking station scene, joystick control, velocity control, waypoint control, ArUco-based docking, FLS sonar, and USBL experiment setup.

## Workspace Layout

```text
src/
  vortex-stonefish-sim/                  Stonefish scenes, robot models, meshes, sensors
  stonefish_ros2/                        ROS 2 wrapper for the Stonefish simulator
  vortex-stonefish-interface/            Stonefish interface launch/config packages
  mundus_mir_stonefish_docking_controller/  ArUco pose estimation and docking controller
  mundus_mir_vehicle_interfaces/         Blueye control launch files and simulator interface
  mundus_mir_velocity_controller_4dof/    4-DOF velocity controller
  mundus_mir_waypoint_controller/         Waypoint and station-keeping controller
  mundus_mir_blueye_joystick_cpp/         Joystick teleoperation node
  mundus_mir_msgs/                       Custom ROS 2 messages and services
```

Important Stonefish files:

```text
src/vortex-stonefish-sim/stonefish_sim/scenarios/docking.scn
src/vortex-stonefish-sim/stonefish_sim/drones/blueye.scn
src/vortex-stonefish-sim/stonefish_sim/objects/docking_station_rig.scn
```

## Build

From the repository root:

```bash
cd ~/Desktop/auto-pilot
source /opt/ros/humble/setup.bash
colcon build
source install/setup.bash
```

After editing `.scn`, launch, or Python/C++ package files, rebuild the affected packages or rebuild the workspace.

## Run the Stonefish Docking Scenario

Terminal 1:

```bash
cd ~/Desktop/auto-pilot
source /opt/ros/humble/setup.bash
source install/setup.bash
ros2 launch stonefish_sim simulation.launch.py scenario:=docking
```

This starts the Stonefish simulator with the Blueye ROV and docking station.

## Run the Docking Stack

Terminal 2:

```bash
cd ~/Desktop/auto-pilot
source /opt/ros/humble/setup.bash
source install/setup.bash
ros2 launch mundus_mir_stonefish_docking_controller stonefish_docking.launch.py debug_view:=true
```

The docking launch starts the ArUco detector, sensor republisher, EKF, docking controller, velocity controller, and Blueye simulator interface/thruster allocation chain.

Useful checks:

```bash
ros2 topic echo /blueye/docking_station_detected --once
ros2 topic echo /blueye/pose_estimated_board_stamped --once
ros2 topic echo /odometry/filtered --once
ros2 topic echo /blueye/ref_vel --once
ros2 topic echo /blueye/thrusters --once
```

## Joystick Control

Use this when manually testing motion and thruster behavior:

```bash
cd ~/Desktop/auto-pilot
source /opt/ros/humble/setup.bash
source install/setup.bash
ros2 launch mundus_mir_vehicle_interfaces stonefish_blueye_control.launch.py
```

The joystick path publishes desired velocity to `/blueye/ref_vel`, which is converted by the velocity controller to `/blueye/cmd_force`, then allocated to `/blueye/thrusters`.

## Waypoint Control

Run the waypoint controller stack, then add/run waypoints using the services exposed by `mundus_mir_waypoint_controller`.

Status check:

```bash
ros2 service call /blueye/get_waypoint_status mundus_mir_msgs/srv/GetWaypointStatus "{}"
```

Run/stop controller:

```bash
ros2 service call /blueye/run_waypoint_controller mundus_mir_msgs/srv/RunWaypointController "{run: true}"
ros2 service call /blueye/run_waypoint_controller mundus_mir_msgs/srv/RunWaypointController "{run: false}"
```

## Sensors

The Blueye model includes camera, IMU, odometry, DVL, FLS sonar, and experimental USBL communication devices.

FLS topics:

```text
/blueye/fls/image      raw sonar intensity image
/blueye/fls/display    colorized display image
```

USBL topics:

```text
/blueye/usbl
/blueye/usbl/beacon_info
```

If Stonefish custom message types fail to echo, make sure this workspace is sourced:

```bash
source ~/Desktop/auto-pilot/install/setup.bash
```

## Notes

- `build/`, `install/`, and `log/` are generated by `colcon` and should not be committed.
- If the local folder path is renamed, do a clean rebuild because generated setup files can contain absolute paths.
- If `/clock` has no publisher, do not run timer-based nodes with `use_sim_time:=true`.
